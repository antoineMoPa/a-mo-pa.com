Introduction à Git

  • Collaborer avec d'autres programmeurs sur une base de code.
  • Avoir un outil pour gérer le code, les ajouts/modification aux code.
  • Éviter de faire manuellement des merges de base de code

Distribué vs Centralisé

La popularité de Git est due au fait qu'il est distribué:


Centralisé

Distribué


Un seul repository/serveurs

  1. George édite son fichier
  2. George envoit son fichier sur le serveur
  3. La famille à George va updater le fichier que George a modifié à partir du serveur

Plusieurs repository/serveurs

  1. George édite son fichier
  2. George édite son fichier et le pousse sur son repo Github/GitLab/son propre serveur
  3. Ginette tire(pull) les modifs de George et les pousse sur son serveur (ex.: git.ginette.com/projet)
  4. Puisque Ginette code bien, la famille lui fait confiance et pull son code et utilise sa version
